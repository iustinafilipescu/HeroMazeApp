1 Motivatie

Marvel Entertainment este una dintre cele mai proeminente companii de divertisment din lume bazate pe personaje, construită pe o bibliotecă de peste 8.000 de personaje din benzi desenate si filme.

DC Comics este una dintre cele mai mari și mai vechi companii americane de benzi desenate. Majoritatea publicațiilor sale au loc în Universul DC fictiv și prezintă numeroase personaje eroice din punct de vedere cultural, precum Superman, Batman și Wonder Woman. Universul are, de asemenea, supravegheați cunoscuți care se opun super-eroilor precum Lex Luthor, The Cheetah și Joker.

Hero Maze doreste sa ofere utilizatorilor interesati de supereroii Marvel si DC un produs placut vizual care sa imbine cunostintele despre mediul inconjuartor cu pasiunea pentru filme intr-un mod educativ si interactiv.

2 Descriere generala

2.1 Modul de functionare a jocului

Este necesara logarea utilizatorului pentru a accesa aplicatia. Jocul este prevazut pe nivele in functie de punctajul acumulat. In cadrul fiecarei partide a jocului, utilizatorul ia identitatea unui supererou pe care acesta il alege din lista supereroilor disponibili la nivelul curent.

Jocul contine o harta pe care sunt localizate opriri. Pentru a avansa de la o oprire la alta, este necesar ca utilizatorul sa raspunda corect la intrebarea curenta. La inceperea jocului, utilizatorul dispune de 3 vieti (fiecare intrebare la care raspunde gresit implica pierderea unui vieti).

Utilizatorul castiga jocul curent in momentul in care trece prin toate opririle si mai are macar o viata. In caz contrat, sesiunea jocului curent se incheie si utilizatorul nu colecteaza niciun punct. Punctele vor fi alocate in functie de numarul de vieti de care utilizatorul dispune la finalul jocului.

2.2 Componente

Login page

  1. Sign in
    • contine input-urile username si password prin intermediul carora utilizatorul poate intra in cont
    • contine un buton care va deschide o fereastra ce ofera descrierea jocului
  2. Sign up
    • contine un formular prin intermediul caruia un utilizator isi poate crea cont, ce contine input-urile: nume, prenume, username, parola si confirmare parola

Home page

  1. Navbar
    • contine meniul aplicatiei prin intermediul caruia utilizatorul poate accesa alte pagini
  2. Hero box
    • contine lista de supereroi disponibili in functie de nivelul utilizatorului
  3. Ranking box
    • contine top 5 utilizatori si punctajele aferente acestora, precum si optiunea de vizualizare a topului general ca feed RSS

Profile page

  1. Navbar
    • contine meniul aplicatiei prin intermediul caruia utilizatorul poate accesa alte pagini
  2. Profile box
    • contine poza de profil (cu posibilitatea schimbarii acesteia)
    • contine statistici despre utilizator (level, wins, lost)
  3. User-Info box
    • contine informatii despre utilizator (nume, prenume, username)
    • ofera posibilitatea schimbarii parolei

Instructions page

  1. Description box
    • contine descrierea si regulile jocului

Training page

  1. Navbar
    • contine meniul aplicatiei prin intermediul caruia utilizatorul poate accesa alte pagini
  2. Question box
    • contine intrebarea si cele 3 variante de raspuns
    • pentru a naviga intre intrebari aplicatia pune la dispozitie sageti pentru next queastion si previous question

Game page

  1. Game map
  2. “Show question” button
    • deschide o fereastra care afiseaza intrebarea curenta, cele trei variante de raspuns si butonul de submit
  3. Info box
    • contine informatii cu privire la scorul utilizatorului, vietile ramase in cadrul jocului curent, nivelul general al utilizatorului si eroul cu care acesta parcurge jocul curent

Results page

  1. Result Box
    • contine intrebarile la care a raspuns utilizatorul pe parcursul jocului, cele rosii fiind gresite iar cele verzi fiind corecte, impreuna cu rezultatul final (win sau lost).
    • in cazul win se precizeaza si numarul de puncte adunate
  2. "Go back home" button
    • buton de intoarece la pagina Home

Admin page

  1. Users Box
    • contine toti utilizatorii aplicatiei impreuna cu usernameul, first name si last name si optiunea de delete pentru fiecare utilizator in parte.
  2. Questions Box
    • contine toate intrebarile din joc impreuna cu variantele de raspuns si varianta corecta.

Feed RSS

  1. Descriere
    • contine topul general al utilizatorilor in functie de wins si este diponibil ca fisier fisier xml sau ca pagina (rss reader)

3 Tipuri de utilizatori

3.1 Utilizatorul neautentificat

  • Poate sa isi creeze un cont de pe pagina de register
  • Poate sa vada instructiunile (pe pagina de Register/Log In este disponibil un pop-up)

3.2 Utilizatorul autentificat

    3.2.1 Utilizator fara drept de administrator

  • Vizualizare date personale (nume, prenume, username, level)
  • Schimbarea parolei si a pozei de profil
  • Se antreneaza pentru joc raspunzand la intrebari
  • Poate selecta un supererou in functie de nivelul la care se afla. Cu acest supererou se poate juca pe pagina de Game

  • Are toate drepturile unui utilizator fara drepturi de administrator
  • Poate accesa pagina Admin care afiseaza toti utilizatorii, cu posibilitatea stergerii acestora
  • Poate accesa pagina Admin care afiseaza toate intrebarile si raspunsurile aferente acestora
  • Poate exporta intrebarile in format CSV sau XML
  • Poate importa in tabela users un fisier CSV

4 External Features

4.1 User Interefaces

Aplicatia web este alcatuita din 8 pagini, 7 pe care orice utilizator logat le poate accesa, una reprezentand modulul de administrare disponibil doar administratorului.

Singura pagina disponibila pentru noii clienti ai aplicatiei este cea de Log in unde au posibilitatea sa isi creeze un cont. De asemenea aceasta pagina afiseaza sub forma unui ferestre de tip pop up informatii referitoare la joc si la scopul acestuia. Tot in aceasta pagina utilizatorii se pot loga, urmand sa fie redirectionati catre pagina Home a aplicatiei. De asemenea aceasta pagina prezinta keyboard shortcuts astfel ca dupa introducerea datelor se poate apasa butonul de Log in sau tasta Enter.

Pagina Home prezinta un meniu de unde utilizatorii pot naviga catre celelalte pagini ale aplicatiei: Profile, Training, Instructions si pagina de Log in ( include delogarea). De asemenea tot de aici se poate accesa jocul, selectand un supererou apasand pe cele 2 sageti sau folosind keyboard shortcuts (sagetile stanga – dreapta). In momentul in care utilizatorul a ales un personaj poate incepe jocul apasand butonul play sau folosind tasta Enter. In partea din dreapta a paginii se regaseste rankingul celor mai buni jucatori si punctajul corespunzator impreuna cu statistica victoriilor/infrangerilor utilizatorului logat. De asemenea topul complet poate fi vizualizat ca feed RSS.

In sectiunea Profile, utilizatorul isi poate actualiza poza de profil si schimba parola.

In sectiunea Training utilizatorul are parte de un preview al jocului antrenandu-se cu intrebarile din joc. Fiecare intrebare are mai multe variante iar in momentul in care utilizatorul alege una din ele acestea se vor colora in functie de validare ( verde – varianta corecta, rosu – variante incorecte). Utilizatorul poate naviga prin intrebari folosind sagetile din pagina.

Pagina Instructions ofera informatii referitoare la joc.

In Game, in partea din stanga, utilizatorul va vedea in timp real punctajul lui pe masura ce raspunde la intrebari impreuna cu supereroul ales in sectiunea anterioara. In partea dreapta acesta va putea vizualiza harta in timp ce avanseaza in joc. Pentru a raspunde la intrebari utilizatorul va apasa butonul Show question urmand a se deschide o fereastra de tip pop up cu intrebarea impreuna cu variantele de raspuns. Dupa ce a ales varianta acesta va trimite raspunsul folosind butonul Send.

Pagina Admin este disponibila doar utilizatorul, putand fiind accesata din meniu (buton vizibil doar atunci cand se conecteaza cu un cont de admin). Aceasta este impartita in 2 sectiuni, Users si Questions. In users sunt afisati toti utilizatorii (username, last name, first name) si butonul de delete user pentru fiecare utilizator in parte care sterge utilizatorul din baza de date. Sectiunea questions afiseaza toate intrebarile din baza de date si variantele de raspuns impreuna cu cea corecta.

Pagina Results este accesata automat atunci cand utilizatorul termina un joc. Aceasta afiseaza intrebarile la care utilizatorul a reusit sa raspunda, acestea fiind colorate in rosu daca s-a raspuns gresit si verde daca s-a raspuns corect. In plus, se afiseaza si mesajul "You lost the game" in cazul unui joc in care utilizatorul a ramas fara vieti sau mesajul "You won the game" impreuna cu punctele adunate in cazul in care jucatorul a terminat jocul fara sa isi piarda toate vietiile.

Paginile pentru RSS se pot accesa din Home de catre orice utilizator logat. Sunt doua variante de vizualizare: ca fisier xml sau prin intermediul interpretorului pentru RSS. Continutul lor consta in topul general al utilizatorilor in functie de numarul de victorii. In pagina este afisat fiecare utilizator cu usernameul sau si numarul de wins.

4.2 Software interfaces

Aplicatia va utiliza o baza de date cu care va comunica pentru a retine datele despre fiecare utilizator in parte si pentru a oferi informatii utilizatorilor. Astfel ca la fiecare creare de cont, baza de date va inregistra datele oferite de utilizator .

Sectiunea Training si Game apeleaza la baza de date pentru a valida raspunsul oferit de utilizatori, ulterior si modificarea punctelor, victoriilor, infrangerilor si nivelului in cazul sectiunii Game. Intrebarile impreuna cu variantele de raspuns si varianta corecta sunt stocate in baza de date.

De asemenea pentru fiecare utilizator se va retine nivelul impreuna cu numarul de victorii si infrangeri, astfel ca sectiunea Ranking din pagina Home va interoga baza de date pentru a afisa statisticile.

In pagina Profile, utilizatorul isi poate schimba parola urmand ca bazei de date sa ii fie actualizat campul care retine parola pentru utilizatorul respectiv, la fel si pentru poza de profil, in plus poza va fi stocata intr-un folder destinat pozelor de profil ale utilizatorilor.

Pagina Results va afisa punctele obtinute pe parcursul unui joc apeland la campul points din baza de date corespunzator utilizatorului.

Pagina Admin comunica cu baza de date afisand intrebarile din tabela questions si variantele de raspuns, si de asemenea permite operatii de delete in tabela users.

Paginile pentru RSS comunica cu baza de date pentru a prelua utilizatorii ordonati descrescator dupa numarul de wins pentru a putea realiza topul.

4.3 Communication Interfaces

Din punct de vedere a comunicarii client - server, implementarea aplicatiei va recurge la tehnici de prevenire a atacurilor precum SQL Injection.

Importul si exportul de date se va realiza folosind formate deschise precum fisierele de tip CSV si JSON ( API ) .

Accesarea datelor se va face prin intermediul protocolului HTTP.

6. Scenarii Use-Case

6.1 Utilizatorul nelogat doreste sa isi creeze un cont

  • Acceseaza aplicatia prin pagina principala
  • In cadrul paginii selecteaza optiunea de Register si completeaza datele necesare
  • Accesarea butonul Submit poate genera doua cazuri: succes (utilizatorul este redirectionat pe pagina de Log In) sau eroare (daca parola confirmata nu corespunde cu cea initiala)

6.2 Utilizatorul nelogat doreste sa isi acceseze contul

  • Acceseaza aplicatia prin pagina principala
  • In cadrul paginii selecteaza optiunea de Log In si completeaza datele necesare
  • Accesarea butonul Login poate genera doua cazuri: succes (utilizatorul este redirectionat pe pagina de Home) sau eroare (parola nu corespunde utilizatorului sau utilizatorul nu este inregistrat)

6.3 Utilizatorul logat doreste sa inceapa o sesiune de joc

  • Acceseaza pagina Home
  • Prin intermediul sagetilor selecteaza supereroul dorit din lista celor disponibili la nivelul curent al utilizatorului
  • Accesarea butonul Play va redirectiona utilizatorul catre pagina Game

6.4 Utilizatorul logat doreste sa isi schimbe parola

  • Acceseaza pagina Profile
  • In cadrul paginii selecteaza optiunea de schimbare a parolei si completeaza parola noua
  • Accesarea butonul Submit poate genera doua cazuri: succes (parola a fost schimbata) sau eroare (parola noua confirmata nu corespunde cu cea noua initiala sau parola veche este introdusa gresit)

6.5 Utilizatorul logat doreste sa isi schimbe poza de profil

  • Acceseaza pagina Profile
  • In cadrul paginii selecteaza optiunea de schimbare a pozei de profil unde se va deschide o fereastra de tip upload
  • O data ales un fisier cu un format specific unei fotografii, acel fisier va fi salvat in folderul aplicatiei destinat pozelor de profil si introdus numele fotografiei in baza de date la campul picture corespunzator utilizatorului in cauza.

6.6 Utilizatorul logat doreste sa se antreneze

  • Acceseaza pagina Training
  • Pe pagina Training este pusa la dispozitia utilizatorului cate o intrebare de cultura generala din baza de date
  • Selectia uneia dintre cele 3 variante de raspuns va informa utilizatorul de corectitudinea raspunsului
  • Utilizatorul are optiunea de a da skip la intrebari

6.7 Utilizatorul logat doreste sa se joace

  • Acceseaza pagina Game prin modul descris la punctul 6.3
  • La prima accesare a paginii utilizatorul trebuie sa deschida pop-up-ul de vizualizare a intrebarii
  • Utilizatorul selecteaza varianta de raspuns, iar ca urmare, in functie de corectitudinea raspunsului, se actualizeaza numarul de puncte sau de vieti. La sfarsitul jocului (ramane fara vieti sau raspunde la toate intrebarile), este redirectionat catre pagina Results care afiseaza intrebarile la care utilizatorul a raspuns colorate aferent (rosu/verde)

6.8 Utilizatorul cu rol de administrator doreste sa acceseze pagina Admin

  • Acceseaza pagina Admin care nu este disponibila pentru utilizatorii obisnuiti
  • Accesarea butonului Export va descarca un fisier de tip CSV cu informatii despre toti userii
  • In cadrul paginii are optiunea de a vizualiza/sterge utilizatori
  • Accesarea butonul Delete din dreptul unui anumit utilizator va duce la stergerea contului acesta

7.3.1 Diagrama arhitectura aplicatiei

In prima diagramă UML se poate observa la nivel înalt modul de funcționare al aplicației. Aceasta este dezvoltată pe baza modelului arhitectural model-view-controller.

Controller-ul se ocupă cu funcționalitatea logică a aplicației. Dacă controller-ul dorește să obțină sau să modifice datele salvate in aplicație, acest lucru se va realiza prin intermediul model-ului corespunzător intrucat modelul are acces asupra bazei de date a aplicației.

View-ul este componenta care primește datele ce urmează a fi expuse din partea unui controller și le afișează conform structurii proprii.

Clasa App are rolul de a ruta cererile către controller-ul corect, precum și către metoda corespunzătoare.

7.4.2. Explicații pentru schema bazei de date

  • Users

    • Tabela Users stocheaza toate informatiile despre utilizatori. De fiecare data cand un nou client isi face cont, datele oferite de acesta ( first name, last name, username, password) sunt inserate in tabela, in plus i se ofera un id (camp care se incrementeaza automat).

    Atributul level reprezinta nivelul la care este utilizatorul, points retine numarul de puncte adunate pe parcursul unui joc, wins si lost contin numarul de jocuri castigate si pierdute (dupa atributul wins se face si clasamentul), question_id si game_question_id reprezinta numarul intrebarii la care a ajuns jucatorul (in sectiunea training si game), picture contine numele fisierului care reprezinta poza de profil a utilizatorului, iar campul isAdmin este 1 doar pentru contul de administrator.

  • Questions

    • Tabela Questions retine toate intrebarile din aplicatie care se afla in sectiunea training si in sectiunea game. Astfel ca fiecare intrebare are un id (acesta fiind cheia primara) si apartine de un caracter intrucat intrebarile difera pentru fiecare supererou in parte (in functie de ce erou a ales utilizatorul pentru a juca un meci). Campul question retine texul intrebarii, iar atributele A_answer, B_answer si C_answer reprezinta variantele de raspuns. Varianta corecta se afla in campul correct_answer.

  • Characters

    • Tabela Characters retine datele despre supereroi, asadar fiecare erou are un id (cheia primara), un nume, si apartine de o companie ( Marvel sau DC).

8. Ghid de utilizare

Pagina login - servește drept pagină de autentificare. Oricine o poate accesa. Din aceasta pagina userii pot doar sa se autentifice si sa vada descrierea jocului.

Pagina home - este pagina prinicpala, de unde se poate ajunge pe toate celelalte pagini, se poate vedea rankingul userilor si se poate incepe jocul.

Pagina game - este pagina pe care playerul logat efectiv se joaca, raspunde la intrebari. Poate veda cate vieti mai are, numarul de puncte, eroul cu care joaca si level-ul la care a ajuns.

Pagina profile - pune la dispoziția utilizatorului o parte din informatiile pe care acesta le are in cont: nume, prenume, username, poza de profil. De asemenea, playerul poate vedea statistici cu privire la progresul lui in joc, dar si poate schimba parola contului si poza de profil.

Pagina instructions - pune la dispoziția utilizatorului o scurta descriere a jocului, de unde acesta poate invata regulile jocului si scopul principal al aplicatiei.

Pagina training - pune la dispoziția utilizatorului un mediu de invatare unde poate raspunde la intrebari pentru a se antrena si a putea sa invete informatii pe care le va utiliza cand se joaca;

Pagina results - afiseaza intrebarile la care a raspuns utilizatorul (cu verde cele corecte, cu rosu cele gresite) si mesajul de victorie (impreuna cu punctele adunate) sau infrangere.

Pagina readrss - afiseaza topul general al utilizatorilor in functie de wins.

Pagina feedrss - afiseaza topul in format XML.

Pagina admin - permite administratorului sa stearga utilizatori din baza de date si sa vizualizeze intrebarile impreuna cu raspunsul corect.
